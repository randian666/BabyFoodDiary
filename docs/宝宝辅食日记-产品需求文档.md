# 宝宝辅食日记｜产品需求文档（PRD）

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 泡泡的辅食日记 |
| 版本 | V1.0 |
| 文档状态 | 可进入产品评审与研发拆分 |
| 依据 | 静态视觉稿、设计规格与视觉 QA 记录 |

## 1. 产品概述

“泡泡的辅食日记”是一款移动端宝宝辅食记录工具，帮助父母快速记录每餐、回顾宝宝对菜谱的接受度，并在满足条件时提示可少量重新尝试的熟悉食材。

产品定位为温暖、轻量的育儿日记，不提供医疗诊断、过敏结论或营养处方。

### 1.1 用户目标

1. 用较少步骤完成一餐辅食记录。
2. 快速了解近期吃过什么、宝宝反应如何。
3. 发现超过 14 天未食用且没有近期风险标记的食材或菜谱。

### 1.2 成功标准

1. 用户能从首页、菜谱库或全局新增按钮进入“记录一餐”。
2. 每条保存成功的记录均包含日期、餐次、至少一道菜谱/食物和整体反应。
3. 首页仅展示满足“好久没吃”规则的复吃推荐。
4. 五个核心页面均可在 390 × 844 的移动视窗内完成核心任务浏览或操作。

## 2. 用户与使用场景

**目标用户：** 正在给婴幼儿添加或记录辅食的父母/照护者。

| 场景 | 用户诉求 | 产品响应 |
| --- | --- | --- |
| 餐后记录 | 迅速记下宝宝吃了什么、反应如何 | 提供预选餐次、菜谱选择、反应选择与保存 |
| 选择复吃食材 | 找到一段时间没吃、可谨慎重试的食材 | 首页展示“好久没吃的食材”推荐卡 |
| 查找菜谱 | 按关键词或类型筛选菜谱并快速记录 | 菜谱库提供搜索、分类、卡片快捷加入 |
| 回顾状况 | 了解近期反应与常吃菜谱 | 提供接受度分析和按日期聚合的历史记录 |

## 3. 信息架构

```text
首页 ──┬── 好久没吃推荐 → 记录一餐（预填食材/菜谱）
       ├── 中央新增按钮 → 记录一餐
       ├── 菜谱
       ├── 分析
       └── 历史

菜谱 ── 菜谱卡“加入本餐” → 记录一餐（预填菜谱）
```

底部固定导航包含“首页、菜谱、分析、历史”四个入口；中央悬浮“+”作为全局新增记录入口。

## 4. 功能需求

### 4.1 首页

首页应让用户一眼确认当天记录进度、可复吃推荐和近期需要关注的情况。

| 编号 | 需求 | 验收标准 |
| --- | --- | --- |
| H-01 | 显示标题“泡泡的辅食日记”与辅助文案“今天也慢慢尝一点”。 | 首屏可见完整标题和辅助文案。 |
| H-02 | 显示“今日辅食概览”，含“重点观察、好久没吃、记录餐次”三项。 | 三项均展示当前统计值；示例状态可为“无、2 种、1 餐”。 |
| H-03 | 显示“好久没吃的食材”模块及推荐卡。 | 卡片至少显示食材名、距上次食用天数、“安全复吃”标签和“记录这次”入口。 |
| H-04 | 推荐卡展示谨慎说明。 | 文案为同等含义的“最近无过敏记录，可少量搭配熟悉菜谱”，不得承诺医学安全。 |
| H-05 | 显示最近常吃的菜谱。 | 每项展示菜谱名、记录次数与最近反应；视觉权重低于推荐卡。 |
| H-06 | 显示重点观察状态。 | 存在观察项时展示相关食材/异常；无数据时显示“这几天没有需要特别观察的食材”。 |

### 4.2 菜谱库

| 编号 | 需求 | 验收标准 |
| --- | --- | --- |
| R-01 | 展示“菜谱库”标题与总数量。 | 例如显示“12 道”。 |
| R-02 | 支持按菜谱名或食材关键词搜索。 | 输入关键词后列表仅保留匹配菜谱或食材的结果。 |
| R-03 | 支持分类筛选。 | 至少包含：全部、粥/糊、软饭、果蔬泥、面点、蒸煮。 |
| R-04 | 菜谱卡展示完整摘要。 | 包含菜谱名、上次记录、记录次数、类型、状态标签和“加入本餐”。 |
| R-05 | 支持快捷记录。 | 点击“加入本餐”进入新增记录页，且该菜谱已预填。 |

### 4.3 接受度分析

| 编号 | 需求 | 验收标准 |
| --- | --- | --- |
| A-01 | 默认展示最近 14 天数据。 | 页面明确显示“最近 14 天”时间范围。 |
| A-02 | 展示总体反应比例。 | 至少包含喜欢、一般、拒绝、过敏的次数和占比；总记录数可见。 |
| A-03 | 展示菜谱接受度排行。 | 每项包含菜谱名、接受度条、记录次数与最近反应。 |
| A-04 | 过敏状态需明确但克制。 | 使用低饱和暖红和文字标签；不得使用恐吓式图形、文案或结论。 |

### 4.4 历史记录

| 编号 | 需求 | 验收标准 |
| --- | --- | --- |
| L-01 | 默认展示最近 14 天记录，并提供时间筛选入口。 | 筛选当前值和“最近 14 天数据”说明清晰可见。 |
| L-02 | 按日期聚合记录。 | 每组显示相对日期和日期，例如“今天 · 07月10日”。 |
| L-03 | 显示餐次记录卡。 | 卡片包含餐次、菜谱名、整体反应、图片缩略图/插画和编辑入口。 |
| L-04 | 支持编辑历史记录。 | 点击编辑可进入带现有数据的新增记录表单。 |

### 4.5 记录一餐

| 编号 | 需求 | 验收标准 |
| --- | --- | --- |
| M-01 | 提供日期选择。 | 默认当天；用户可修改日期。 |
| M-02 | 提供餐次选择。 | 可选早餐、午餐、晚餐、加餐；默认按当前时间推断，无法推断时可默认午餐。 |
| M-03 | 支持添加菜谱/食物。 | 可从已有菜谱选择，并支持“添加自定义菜谱”。 |
| M-04 | 支持可选照片。 | 用户可拍摄或从相册选择；未添加照片不影响保存。 |
| M-05 | 支持整体反应选择。 | 可选喜欢、一般、拒绝、过敏；默认可为“喜欢”。 |
| M-06 | 支持可选备注。 | 可录入自由文本，例如食量、反应或饮食情况。 |
| M-07 | 保存前校验。 | 未添加任何菜谱/食物时，提示“至少添加一道宝宝尝过的食物”，且不可保存。 |
| M-08 | 支持带入上下文。 | 从首页推荐进入时预填推荐食材或关联菜谱；从菜谱库进入时预填所选菜谱。 |
| M-09 | 保存结果可回溯。 | 保存成功后，记录应进入对应日期的历史记录，并参与首页概览及接受度统计。 |

## 5. 关键业务规则

### 5.1 菜谱状态

| 状态 | 判定规则 | 展示建议 |
| --- | --- | --- |
| 新菜谱 | 从未被记录过。 | 淡黄标签 |
| 好久没吃 | 曾记录过，且距最近一次记录超过 14 天。 | 浅绿或浅桃标签 |
| 常吃 | 最近 14 天内多次记录，且无异常反应。 | 正向状态标签 |
| 观察中 | 新食材或异常反应发生后 72 小时内。 | 暖黄色提示 |

### 5.2 复吃推荐规则

仅当食材或菜谱同时满足以下条件时，才可进入首页“好久没吃的食材”推荐：

1. 曾有至少一条历史记录；
2. 距离最近一次记录 **至少 14 天**；
3. 当前不在 72 小时重点观察窗口内；
4. 最近记录未标记为“过敏”。

“安全复吃”仅为界面状态标签，不构成医学安全保证。所有相关文案须使用“适合少量重新尝试”“最近无过敏记录”等谨慎表述。

### 5.3 接受度统计

1. 统计窗口默认最近 14 个自然日，包含当天。
2. 每条有效餐次记录按其整体反应计入喜欢、一般、拒绝或过敏之一。
3. 占比 = 某反应记录数 ÷ 窗口内总有效记录数；总数为 0 时显示空状态，不显示误导性百分比。
4. 菜谱排行默认按接受度由高到低排序；接受度计算口径需在研发阶段确定并保持一致。

## 6. 数据要求

### 6.1 核心实体

| 实体 | 最小字段 |
| --- | --- |
| 菜谱 | ID、名称、分类、关联食材、图片/插画、创建时间 |
| 食材 | ID、名称、最近记录日期、是否处于观察期、最近是否过敏 |
| 餐次记录 | ID、日期、餐次、菜谱/食物列表、整体反应、照片、备注、创建/更新时间 |

### 6.2 枚举值

- 餐次：早餐、午餐、晚餐、加餐。
- 整体反应：喜欢、一般、拒绝、过敏。
- 菜谱分类：全部、粥/糊、软饭、果蔬泥、面点、蒸煮。

## 7. 体验与视觉要求

1. 以 390 × 844 移动视窗为设计基准，首屏展示各页面核心任务。
2. 使用奶油背景 `#FFF9F5`、杏色主操作 `#FF9B7B`、浅绿正向状态 `#7EC8A3`；过敏仅使用低饱和暖红。
3. 卡片使用约 24px 圆角、低对比阴影和暖灰描边，避免冷色调和医疗化表达。
4. 底部导航固定可见，当前项使用杏色；中央新增按钮使用杏色并具有高辨识度。
5. 使用统一的柔和食物插画或餐盘缩略图；不得使用水印、乱码、被裁切的素材或占位图。
6. 文案短、具体、易被父母理解；不输出诊断、风险保证或营养处方。

## 8. 本期范围与非目标

**本期范围：** 五个核心页面、菜谱浏览与筛选、餐次新增/编辑、历史查看、最近 14 天接受度统计、72 小时观察提示、14 天复吃推荐。

**非目标：**

1. 登录、账号体系、宝宝档案及家庭协作；
2. 云同步、后端服务、多端数据共享；
3. 医疗诊断、过敏风险判定或营养建议；
4. 复杂图表、高级筛选和可点击原型交付；
5. 静态视觉稿之外的设计系统扩展。

## 9. 发布验收清单

- [ ] 首页可见今日概览、14 天复吃推荐、最近常吃和重点观察。
- [ ] 推荐结果均通过 14 天、72 小时观察和最近过敏状态校验。
- [ ] 菜谱库支持搜索、分类和“加入本餐”预填。
- [ ] 新增记录支持日期、餐次、菜谱、照片、反应、备注和保存校验。
- [ ] 保存或编辑后，历史、首页统计和分析数据一致更新。
- [ ] 分析页可展示最近 14 天四类反应比例与菜谱排行。
- [ ] 历史页按日期分组，卡片可展示餐次、菜谱、反应、缩略图和编辑入口。
- [ ] 全部页面符合暖色视觉、可读性和无医疗承诺要求。
