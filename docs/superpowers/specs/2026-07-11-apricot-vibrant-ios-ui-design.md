# 泡泡的辅食日记：暖阳橘 UI 设计说明

## 目标

新建一个 iOS 17+ SwiftUI App，将 `style-variants/05-apricot-vibrant-full.html` 的暖阳橘活力风落地为可浏览的原生界面。本期仅交付静态界面与示例数据，不连接存储、网络或真实业务逻辑。

## 范围

应用以 390 × 844pt iPhone 为主要适配目标，包含以下五个界面：

1. 首页：宝宝信息卡、今日概览、复吃菜谱横向卡片与观察提示。
2. 菜谱库：搜索样式、分类标签、排序行与菜谱列表。
3. 接受度分析：14 天反应环形图、统计标签与排行卡片。
4. 历史记录：时间筛选、按日期分组的餐次记录卡片。
5. 记录一餐：日期、餐次、已选菜谱、照片占位、反应与备注等表单样式。

底部使用 4 个入口（首页、菜谱、分析、历史）及居中的悬浮“+”按钮。允许这些控件通过本地 `@State` 切换当前展示页，以便完整预览，但不会保存、更改、筛选或校验数据。

## 视觉与组件

- 颜色：奶油底色、杏橘主色、柠檬黄点缀、低饱和绿色/暖红状态色；颜色以共享 SwiftUI token 定义。
- 层级：大圆角白色卡片、柔和暖色阴影、横向标签（pill）与圆角缩略图。
- 图标：使用 SF Symbols，食物缩略图使用渐变底色与原生符号进行抽象表达。
- 结构：按职责拆分应用入口、颜色/样式 token、底部导航、复用卡片及 5 个页面，避免把所有布局集中到单一 View。

## 技术边界

- 平台：SwiftUI，iOS 17+。
- 数据：代码内 fixture，仅支持展示。
- 状态：根视图使用局部 `@State` 管理当前 tab 与是否显示记录页；不创建 ViewModel、持久化层或服务层。
- 验证：使用 `xcodebuild` 编译工程；若可用，再运行 iOS Simulator 截图确认整体视觉。

## 不包含

搜索、筛选、添加、编辑、保存、图片选择、日期选择、数据统计计算、过敏判断、网络、账号与持久化。
