# 泡泡的辅食日记｜设计系统

> **设计方向：糖果乐园（Playful Candy）**
> 以明快、柔软的糖果色营造儿童感，但将阅读、记录和风险提示置于装饰之前。珊瑚橙始终代表主要操作；粉紫、薄荷绿和奶油黄只用于表面层、插画背景与非破坏性状态。

## 1. 原则

1. 记录优先：每页只突出一个主操作，不让插画、渐变遮蔽日期、食材或反应信息。
2. 安心而非医疗化：过敏与观察状态清晰、克制，不使用警报式动画或恐吓性文案。
3. 儿童感来自圆角、轻量插画和糖果色块；不使用厚重 3D 黏土阴影、大面积高饱和渐变或 emoji 作为结构图标。
4. 可读可点：正文不小于 16px，互动目标不小于 44 × 44px，相邻触点至少间隔 8px。

## 3. 布局与导航

- 以 390 × 844pt 为基准；卡片内边距 16px，模块间距 32px。顶端与底端均遵守系统安全区。
- 固定底部导航和底部 CTA 需给滚动内容预留“导航高度 + 24px”内边距；平板最大内容宽度 640px 且居中。
- 底部导航仅有首页、菜谱、分析、历史四个一级入口；中央“+”是全局“记录一餐”，不是导航目的地。
- 图标使用同一套 1.75–2px 线性 SVG，视觉尺寸 24px。当前项同时改变为珊瑚橙、较高字重和选中图标状态；未选中使用 `text.secondary`。

## 4. 组件规范

### 4.1 按钮与卡片

| 组件 | 规范 |
| --- | --- |
| 主按钮 | `primary` 实底、白色 16px/700 文字、16px 圆角、高 48px；按下改为 `primary.pressed` 并缩放至 0.98；禁用 45% 不透明度。 |
| 次按钮 | `surface` 底、`border` 描边、`text.primary`；高 44px，按下使用 `surface.subtle`。 |
| 文字按钮 | 无底色、`primary` 文字，可带 SVG 图标；点击区至少 44px。 |
| 悬浮新增 | 56px 珊瑚橙圆形、白色加号、`elevation.float`；提供“记录一餐”无障碍名称。 |
| 标准内容卡 | `surface`、24px 圆角、1px `border`、`elevation.card`；不嵌套复杂卡片。 |
| 糖果主题卡 | 使用粉紫、淡紫、薄荷或奶油黄之一；每个首屏最多一个高装饰卡，正文仍为深色。 |
| 重点推荐卡 | 奶油黄至淡杏色表面，可配食材插画；安全复吃为薄荷标签，须保留谨慎说明。 |
| 状态标签 | 高 24px、12px 字、12px 圆角、左右 8px 内边距；始终使用图标 + 文字。 |

### 4.2 表单与反馈

| 组件 | 规范 |
| --- | --- |
| 输入与选择器 | 高至少 48px、16px 圆角、可见标签；占位文本不能替代标签。 |
| 分段选择 | 餐次和反应的单项至少 44px 高、8px 间距；选中项同时改变底色、文字和图标。 |
| 搜索框 | 左侧搜索 SVG 图标，标签“搜索菜谱或食材”；清除按钮点击区至少 44px。 |
| 菜谱选择卡 | 缩略图、菜谱名、元数据及明确“加入本餐”动作；图片有描述性替代文本。 |
| 照片区 | 显示拍摄、从相册选择两入口；预留固定比例防布局跳动。 |
| 错误 | 位于字段下方，使用 `danger` 图标和修复提示；提交后焦点移至首个无效项。 |
| 保存反馈 | 保存中禁用重复点击；超过 300ms 显示局部进度；成功后短提示并可触发一次轻触觉反馈。 |
| 弹层 | 24px 顶角、`elevation.sheet`、48% 遮罩；有关闭按钮和下滑关闭，未保存内容关闭前确认。 |
| 空状态 | 小型食材插画、短说明、一个可执行动作；不显示空白容器。 |

## 5. 状态、数据与页面应用

| 状态 | 表达 | 背景 / 前景 |
| --- | --- | --- |
| 喜欢 | 心形 + “喜欢” | `success.soft` / `success` |
| 一般 | 中性表情图标 + “一般” | `warning.soft` / `warning` |
| 拒绝 | 关闭图标 + “拒绝” | `danger.soft` / `danger` |
| 过敏 | 警示三角 + “过敏”，无动效 | `danger.soft` / `danger` |
| 观察中 | 眼睛图标 + “观察中” | `warning.soft` / `warning` |
| 安全复吃 | 叶片或盾牌 + “安全复吃” | `success.soft` / `success` |

| 页面 | 儿童感用法 | 关键约束 |
| --- | --- | --- |
| 首页 | 奶油黄推荐卡、食材插画、薄荷安全标签 | 首屏可见 14 天推荐；附谨慎说明。 |
| 菜谱库 | 粉紫/薄荷选中筛选项、糖果色缩略图底 | 搜索、分类和“加入本餐”优先可读。 |
| 接受度分析 | 柔和色点缀比例条与排行 | 过敏为低饱和暖红；图例与数值同时可见。 |
| 历史记录 | 淡紫日期胶囊、白底餐次卡 | 日期分组和编辑入口清晰。 |
| 记录一餐 | 彩色分段控件、淡粉照片区 | 保存按钮仅用珊瑚橙；标签、必填和错误清楚。 |

分析页不只依赖色块区分反应，使用文字图例与直接数值；无数据时显示“最近 14 天还没有记录”及“记录一餐”入口。

## 6. 动效、无障碍与禁用项

1. 按下、选中、淡入时长 180–240ms，`ease-out`；页面转场不超过 320ms，只动画化 `opacity` 和 `transform`。减少动态效果时取消缩放和位移。
2. 支持动态字体；文字变大时优先换行、增加卡片高度，不裁切菜谱名、状态和错误。
3. 所有图标按钮具备可读名称，读屏顺序遵循视觉顺序；Web 焦点环为 2px `focus` 外描边，Tab 顺序一致。
4. 禁止粉紫担当保存/确认/当前导航；禁止 emoji 充当导航、操作或状态的唯一图标；禁止一个页面有超过一个强渐变或两个高装饰糖果卡；禁止小于 44px 的触点、低对比正文、仅颜色状态或悬停专属操作。
5. 交付检查：375pt 小屏、横屏、系统最大字体、减少动态效果、安全区、底部导航遮挡，以及状态的文字/图标/颜色三重表达。
